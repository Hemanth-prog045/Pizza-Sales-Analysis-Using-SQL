# 🍕 Pizza Sales Analysis using SQL

An end-to-end SQL data analytics project that analyzes Pizza Hut sales data to uncover business insights using MySQL. This project demonstrates SQL proficiency through data exploration, aggregation, joins, subqueries, Common Table Expressions (CTEs), and window functions.

---

## 📖 Project Overview

This project analyzes a Pizza Hut sales dataset to answer real-world business questions related to:

- Customer ordering behavior
- Revenue analysis
- Best-selling pizzas
- Sales trends
- Category-wise performance
- Daily order patterns

The objective is to transform raw transactional data into actionable business insights using SQL.

# 📊 Business Questions Solved

## Basic Analysis

- Retrieve the total number of orders.
- Calculate total revenue generated.
- Identify the highest-priced pizza.
- Find the most common pizza size ordered.
- List the top 5 most ordered pizza types.

---

## Intermediate Analysis

- Join multiple tables to analyze sales.
- Determine category-wise pizza sales.
- Calculate average pizzas ordered per day.
- Identify top-selling pizzas by quantity.
- Analyze revenue generated by each pizza category.

---

## Advanced Analysis

- Calculate percentage contribution of each pizza category.
- Analyze cumulative revenue over time.
- Determine the top 3 revenue-generating pizzas within each category.
- Perform ranking using Window Functions.
- Generate business insights using Subqueries and CTEs.

# 🚀 How to Run the Project

## 1. Clone the repository

```bash
gh repo clone Hemanth-prog045/Pizza-Sales-Analysis-Using-SQL
```

---

## 2. Open MySQL Workbench

Create a database:

```sql
CREATE DATABASE pizzahut;
USE pizzahut;
```

---

## 3. Import the CSV files

Import:

- orders.csv
- orders_details.csv
- pizzas.csv
- pizza_types.csv

using **Table Data Import Wizard**.

---

## 4. Execute SQL Scripts

Run the SQL files in order:

```
01_basic_queries.sql
```

```
02_intermediate_queries.sql
```

```
03_advanced_queries.sql
```
